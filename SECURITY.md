# セキュリティポリシー

## 脆弱性の報告

`jp-pii-detect` に関する脆弱性を見つけた場合は、**公開 issue には書かず**、GitHub の
Private Vulnerability Reporting（Security Advisories）経由で報告してください。

- リポジトリの **Security** タブ → **Report a vulnerability** から非公開で報告できます。

内容を確認のうえ、修正方針や公開時期についてご連絡します。報告いただいた脆弱性は、
修正版のリリースと合わせて Security Advisory として公開します。

## サポート対象バージョン

セキュリティ修正は **最新リリースのみ** を対象とします。古いバージョンを利用中の場合は、
最新版へのアップグレードをお願いします。

## このツールを扱う際の注意

`jp-pii-detect` は個人情報（PII）を検出する性質上、その入出力に機密情報が含まれ得ます。

- 検出結果（`--unmask` 出力、JSON / SARIF レポート）には、検出された PII の値そのものが
  含まれる場合があります。CI ログや成果物として保存する際は取り扱いに注意してください。
  既定では検出値はマスク表示されます。
- ベースラインファイル（`.jp-pii-baseline.json`）は salt 付き HMAC-SHA256 の値ハッシュを
  記録しますが、ファイル自体を入手した第三者による低エントロピー値への総当たり照合まで
  防ぐものではありません。元のソース履歴と同程度の機密度で扱い、公開リポジトリには
  コミットしないでください（詳細は [README](README.md) の「ベースライン」節を参照）。
- issue や pull request、テストケースには、実在する PII を貼らずに必ずダミー化してください。
