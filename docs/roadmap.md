# v1.0 ロードマップと安定性ポリシー

`jp-pii-detect` の今後の方向性と、後方互換に関する方針をまとめます。

## v1.0 の定義（後方互換の保証範囲）

v1.0 に到達した時点で、以下を **後方互換の保証対象** とします。これらを破壊する変更は、
メジャーバージョンを上げない限り行いません。

- **ルール ID**: `jp-my-number` などのルール ID は、既存のものを勝手にリネーム・削除しません。
- **`.jp-pii.toml` 設定フォーマット**: 既存のキー・セクションの意味を変えません。
- **終了コード**: `0`（検出なし）/ `1`（検出あり）/ `2`（エラー）の割り当てを維持します。
- **JSON 出力スキーマ**: 既存フィールドの削除・意味変更を行いません（フィールドの追加は互換の範囲）。

## v1.0 までの破壊的変更ポリシー

v1.0 到達前も、変更は無秩序には行いません。

- **検出結果が変わる変更（ルール・辞書・正規化の変更）はマイナーバージョンで行い、
  リリースノートに必ず明記します。** どのルールがどう変わるかを記載し、利用者が影響を
  把握できるようにします。
- 精度の実測値（F1 等）が動く変更は、CI の精度ゲート（`docs/accuracy.json`）を更新した
  うえでリリースします。
